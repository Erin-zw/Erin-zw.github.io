Rethinking Semantic Segmentation from a Sequence-to-Sequence Perspective with Transformers

概述

作者认为目前的网络都是基于FCN结构的编码器解码器结构，encoder特征提取增加感受野，decoder上采样进行像素级分类。对于传统CNN无法实现长距离上下文建模，目前有两种解决方式，大尺寸kernel或者空洞卷积，还有就是注意力机制。但作者认为在架构上并没有脱离FCN的范畴，所以作者决定在网络结构上进行改变。

作者使用纯transformer代替分辨率逐渐下降的基于卷积的编码器，编码器trans每一层都没有下采样而是进行全局建模

主要进行了三个工作：

  • (1)从序列到序列学习的角度重新阐述了图像语义分割问题，为占主导地位的编码器-解码器 FCN 模型设计提供了替代方案。
    1. 利用 Transformer 框架通过序列化图像来实现完全基于注意力的特征表示编码器。
  • (3)为了广泛测试自注意力特征表示，介绍了三种不同复杂度的解码器设计。

三种解码器架构

SETR-NAIVE:直接进行分割操作

SETR-PUP:渐进上采样经典U型结构

SETR-MLA:多层级特征融合,算是本文唯一的看点.其中Z6,Z12,Z18,Z24代表了多trans块的中间输出.

旋转图像

本文字数： 738 阅读时长 ≈ 1 分钟

给定一个 n × n 的二维矩阵 matrix 表示一个图像。请你将图像顺时针旋转 90 度。

你必须在原地 旋转图像，这意味着你需要直接修改输入的二维矩阵。请不要 使用另一个矩阵来旋转图像。

我的思路：由外而内，先看第一个外圈的四个角元素，会发现顺时针旋转90°就是把左上和左下交换，左下和右下交换，右下和右上交换。由此旋转完成。接着我们把四个角元素的位置都移动一位，会发现交换原则没变，所以n-1次可以旋转完成最外层。接着向内k层，会发现要n-1-2*k次。我们总共要向内n/2-1层。注意下标即可。

乘法

发表于 2024-03-26 分类于 深度学习
本文字数： 798 阅读时长 ≈ 1 分钟

在 PyTorch 中，常用的矩阵乘法函数有以下几种：

  1. torch.mm():
    • 用于计算两个二维矩阵之间的矩阵乘法。例如，对于大小分别为 (m x n) 和 (n x p) 的两个矩阵，torch.mm(matrix1, matrix2) 返回一个 (m x p) 大小的结果矩阵。
  2. torch.matmul():
    • 是一个更为通用的矩阵乘法函数，它可以处理任意维度的张量，遵照标准的矩阵乘法规则。对于二维矩阵乘法，它的行为与 torch.mm() 相同。对于更高维度的张量，遵循广播规则和矩阵乘法的广义定义。
  3. @ 运算符:
    • 自从Python 3.5版本开始，引入了一个专门用于矩阵乘法的运算符 @。在PyTorch中，你可以直接使用这个运算符来代替 torch.matmul() 进行矩阵乘法。
  4. torch.bmm():
    • 批量矩阵乘法，用于计算两个三维张量间的矩阵乘法，其中最后一个维度被认为是矩阵的内部维度，进行矩阵乘法。
  5. 点乘（逐元素乘法）:
    • 如果你想要进行的是逐元素（element-wise）乘法，而不是矩阵乘法，可以使用 torch.mul() 或者使用 * 运算符。

请注意，对于真正的矩阵乘法（并非逐元素乘法），一定要确保输入的张量满足矩阵乘法的要求，即左矩阵的列数等于右矩阵的行数。

contiguous操作

发表于 2024-03-26 分类于 深度学习
本文字数： 660 阅读时长 ≈ 1 分钟

在PyTorch中，.contiguous() 方法主要用于确保张量的内存布局是连续的，也就是数据在内存中是按顺序存放，没有间隙。通常，在以下几种情况下，需要调用 .contiguous() 方法：

  1. 进行 view() 或 reshape() 操作之前：

    • 若想通过 .view() 或 .reshape() 方法改变张量的形状而不改变其元素数量时，必须确保张量是连续的。如果不连续，这些操作可能会失败或者导致不可预期的结果。例如，在对张量进行转置或通道交换等操作之后，再尝试重塑形状时，往往需要先调用 .contiguous()。

    例如：proj_query.permute(0, 3, 1, 2).contiguous().view(m_batchsize * width, -1, height).permute(0, 2,1)

  2. 需要连续内存布局的操作：

    • 部分高级功能或操作要求输入张量是连续的，比如使用 CUDA 内核进行自定义操作、或者在卷积神经网络中使用某些特殊类型的张量格式。

  3. 涉及索引、切片后的张量：

    • 有时候，对张量进行索引或切片操作后，得到的新张量可能是非连续的，如果后续操作需要连续性，那么就需要先调用 .contiguous()。

  4. 与其他库接口交互时：

    • 当需要将张量传递给依赖于连续内存布局的外部库（如 C 库或其他 Python 库）时，为了保证正确性和效率，可能需要调用 .contiguous()。

简单来说，只要你的代码逻辑中可能出现张量由于某些操作导致内存布局不连续，而后续又有操作要求连续布局的情况下，就应该适时地使用 .contiguous() 来确保内存布局连续。

CCNet

CCNet（Criss Cross Network）的核心是重复十字交叉注意力模块。该模块通过两次CC Attention，可以实现目标特征像素点与特征图中其他所有点之间的相互关系，并用这样的相互关系对目标像素点的特征进行加权，以此获得更加有效的目标特征。

non-local 模型中， 因为要考虑特征图每个像素点与所有像素点的关系，时间复杂度和空间复杂度为 O(HW*HW)。 在CC Attention模块中，计算特征图中每个像素点与其对应的行列的像素点的关系，时间复杂度和空间复杂度O(HW*(H + W - 1)) ,相比前者降低了一个1~2个数量级。

大致流程：输入一张图片，经过CNN网络提取特征输出特征图X，X经过1*1卷积降维，获得特征图H，H经过两个CCA模块。

CCA模块大致流程：

H经过两个1*1卷积降维获得QK，c1>c2，经过一个1*1卷积获得V，V通道数不变。

H: (batch,c1,h,w) #输入特征图 Q: (batch,c2,h,w) #Query查询向量 K: (batch,c2,h,w) #Key 键值向量 V: (batch,c1,h,w) #Value 值向量

获取权重：

  • step1: 取Q中特征图中某一像素点的所有通道值：q = Q(i,j) , size = (1, c2)

  • step2: 取K特征图中与q同一行和同一列的所有像素点的所有通道值, 交叉位置取了两次，只选一次.k的 size = (c2, h+w - 1)

  • step3: q*k, 得到q_atten, size = (1, h+w - 1), 并对这(h + w -1) 个值进行softmax操作，即权重和为1.

  • step4: 对Q中的所有像素点重复step2 和 step3， 即得到了每个像素点的归一权重。此时atten.size = (batch, h,w, h + w - 1)

affinity操作：

上面已经获取了Atten — (batch, h, w, h + w - 1)。接下来将权重施加在V上。 先不考虑batch

  • step1: 获取Atten中某个像素点的所有权重, A = Atten(i,j) , size = (1, h + w -1)

  • step2: 取V的某一通道Cn 的特征图Vn, size = (h, w) , 选取Vn上与A对应位置的同一行和同一列的数值，记作vn，size = (1, h + w - 1)

  • step3: vn 与 A.T 相乘，即得到加权后的vn值，size = (1,1)

  • step4: 对V中的所有通道重复step2 和 step3操作。

  • step5: 对Atten中的所有像素点重复上述4步操作。

  • step6: 残差网络：H‘ = CCAtten(H) + H，size：（batch，c1，h，w）

串联CCA模块：

一个CCA模块输出的特征图只能获得一行一列的信息，但是如果串联另一个CCA模块即可获得全局信息。

如图所示，如果(ux,uy)想获得(θx,θy)的信息，第一次CCA模块只能获得相对应行列的信息，也就是说第一次运算(ux,uy)获得了(ux,θy)和(θx,uy)的信息，但是(ux,θy)，(θx,uy)也获得了(θx,θy)的信息，也就是说再经过一次CCA模块就可以把(ux,uy)，(θx,θy)关联起来，彼此获得信息。也就是说我们使用两个CCA模块就可以达到获得全局信息的效果。

MedTrans

发表于 2024-03-21 更新于 2024-03-26 分类于 深度学习论文
本文字数： 1.2k 阅读时长 ≈ 1 分钟

Medical Transformer: Gated Axial-Attention for Medical Image Segmentation

概述

​ 卷积结构存在内在的inductive biases（归纳偏差），缺乏对图像远程依赖关系的理解。同时对于数据集规要求较高。

​ 作者提出门控轴向注意力机制，并且提出一种叫做Local-Global（LoGo）的训练策略高效训练模型。在三个数据集上达到了更好的效果相比卷积模型和其他注意力模型。

​ 作者认为虽然Deeplab，SETR这些网络提高了表现，但是还未被探索完全，因为编码解码器仍然是以卷积结构作为主要模块。

​ 同时trans结构需要大规模数据集，但是对于医学图像难以做到，并且较少的图像会导致训练位置编码变得困难。所以提出门控的注意力控制位置编码进入qkv的信息量。由于是可学习的，所以可以适用于基本上所有规模的数据集。根据数据集大小，这些门会学习是否图片的数量可以学习到正确的位置编码。根据判断学习到的位置编码是否有用，这些门参数会收敛到0或者一个high value。

​ 提出一种局部全局（LoGo）训练策略，使用浅全局分支和对医学图像的补丁进行操作的深层局部分支。不仅关注整张图片同时还关注局部细节。

总结：

  • 提出了一种门控位置敏感的轴向注意机制，即使在较小的数据集上也能很好地工作
  • 引入LoGo训练方法
  • 提出Med-Trans
  • 提高了表现

实现细节

模型结构：

轴向注意力：

将特征图分为两个维度（长宽轴）进行注意力计算。具有更高的计算效率。

所谓的轴向注意力就是对于位置编码分别对qkv进行学习，然后进行相加。具体看公式： \[ y=\sum softmax(qk+qr_q+kr_k)(v+r_v) \] 门控轴向注意力：

作者认为，对于小规模数据集的实验（医学图像分割中常见的情况），位置偏差很难学习，因此在编码远程交互时并不总是准确的。在学习的相对位置编码不够准确的情况下，将它们添加到各自的键、查询和值张量中将导致性能下降。因此提出门控轴向注意力，也就是加入了门控制位置信息量的输入。公式： \[ y=\sum softmax(qk+G_qqr_q+G_kkr_k)(G_{v_1}v+G_{v_2}r_v) \] LoGo训练策略：

为了提高对图像的整体理解，作者建议在网络中使用两个分支，即作用于图像原始分辨率的全局分支和作用于图像补丁的局部分支。

全局分支中，减少了门控轴向变压器层的数量，因为观察到所提出的变压器模型的前几个块足以模拟长范围依赖性。在本地分支中，创建 16 个大小为 I/4 × I/4 的图像块，其中 I 是原始图像的尺寸。在本地分支中，每个补丁通过网络进行前馈，并根据输出特征图的位置对其进行重新采样以获得输出特征图。然后将两个分支的输出特征图相加并通过 1 × 1 卷积层以生成输出分割掩模。
